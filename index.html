如何部署Docker-MySQL

启动一个 MySQL Docker 容器并配置其基本参数，可以使用以下命令。本文将逐步解释命令中的各个部分。

详细解释

1. 运行权限

**sudo**：以超级用户权限运行 Docker 命令。这通常是为了确保有足够的权限来执行 Docker 命令。

2. 分离模式

**docker run -d**：以分离模式（detached mode）运行一个新的容器。容器将在后台运行。

3. 容器命名

**--name syr-mysql**：为容器指定一个名称为 syr-mysql。这样可以更方便地管理和访问这个容器。

4. 端口映射

**-p 3306:3306**：将主机的 3306 端口映射到容器的 3306 端口。3306 是 MySQL 的默认端口，这样主机可以通过 localhost 的 3306 端口来访问容器内的 MySQL 服务。

5. 时区配置

**-e TZ=Asia/Shanghai**：设置环境变量 TZ，将容器的时区配置为 Asia/Shanghai。

6. 设置 MySQL root 密码

**-e MYSQL_ROOT_PASSWORD=123**：设置环境变量 MYSQL_ROOT_PASSWORD，指定 MySQL 的 root 用户密码为 123。这是 MySQL 初始化时所需的配置。

7. 挂载数据目录

**-v /home/syr/mysql/data:/var/lib/mysql**：将主机目录 /home/syr/mysql/data 挂载到容器的 /var/lib/mysql。这是 MySQL 的数据存储目录，挂载主机目录可以持久化数据，防止容器删除时数据丢失。

8. 挂载配置目录

**-v /home/syr/mysql/conf:/etc/mysql/conf.d**：将主机目录 /home/syr/mysql/conf 挂载到容器的 /etc/mysql/conf.d。这是 MySQL 的配置文件目录，可以在主机上管理 MySQL 的配置文件。

9. 挂载初始化脚本目录

**-v /home/syr/mysql/init:/docker-entrypoint-initdb.d**：将主机目录 /home/syr/mysql/init 挂载到容器的 /docker-entrypoint-initdb.d。在这个目录中的任何 .sql 或 .sh 文件将在 MySQL 容器启动时自动执行，用于初始化数据库。

10. 加入 Docker 网络

**--network syr**：将容器连接到名为 syr 的 Docker 网络中。这样，容器可以与同一个网络中的其他容器通信。

11. 指定 MySQL 镜像

**mysql**：指定使用 mysql 镜像来创建容器。如果没有本地镜像，Docker 会从 Docker Hub 中拉取最新的 mysql 镜像。

总结

以上命令启动了一个 MySQL 数据库容器，配置了时区和 root 密码，并将数据目录、配置目录和初始化脚本目录映射到主机上的相应目录。同时，这个容器被加入到名为 syr 的 Docker 网络中。这样可以确保数据持久化、配置文件可管理化以及初始化脚本的自动执行。
