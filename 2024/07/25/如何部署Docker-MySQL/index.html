如何部署Docker-DySQL

这个命令用于启动一个MySQL Docker容器，并配置它的一些基本参数。让我们逐个解释命令中的各个部分：

详细解释：

  1. sudo：以超级用户权限运行Docker命令。这通常是为了确保有足够的权限来执行Docker命令。
  2. docker run -d：以分离模式（detached mode）运行一个新的容器。容器将在后台运行。
  3. –name syr-mysql：为容器指定一个名称为syr-mysql。这样可以更方便地管理和访问这个容器。
  4. -p 3306:3306：将主机的3306端口映射到容器的3306端口。3306是MySQL的默认端口，这样主机可以通过localhost的3306端口来访问容器内的MySQL服务。
  5. -e TZ=Asia/Shanghai：设置环境变量TZ，将容器的时区配置为Asia/Shanghai。
  6. -e MYSQL_ROOT_PASSWORD=123：设置环境变量MYSQL_ROOT_PASSWORD，指定MySQL的root用户密码为123。这是MySQL初始化时所需的配置。
  7. -v /home/syr/mysql/data:/var/lib/mysql：将主机目录/home/syr/mysql/data挂载到容器的/var/lib/mysql。这是MySQL的数据存储目录，挂载主机目录可以持久化数据，防止容器删除时数据丢失。
  8. -v /home/syr/mysql/conf:/etc/mysql/conf.d：将主机目录/home/syr/mysql/conf挂载到容器的/etc/mysql/conf.d。这是MySQL的配置文件目录，可以在主机上管理MySQL的配置文件。
  9. -v /home/syr/mysql/init:/docker-entrypoint-initdb.d：将主机目录/home/syr/mysql/init挂载到容器的/docker-entrypoint-initdb.d。在这个目录中的任何.sql或.sh文件将在MySQL容器启动时自动执行，用于初始化数据库。
  10. –network syr：将容器连接到名为syr的Docker网络中。这样，容器可以与同一个网络中的其他容器通信。
  11. mysql：指定使用mysql镜像来创建容器。如果没有本地镜像，Docker会从Docker Hub中拉取最新的mysql镜像。

总结：

这个命令启动了一个MySQL数据库容器，配置了时区和root密码，并将数据目录、配置目录和初始化脚本目录映射到主机上的相应目录。同时，这个容器被加入到名为syr的Docker网络中。这样可以确保数据持久化、配置文件可管理化以及初始化脚本的自动执行。
